 Olist E-Commerce Sales Dashboard

 Project Overview

This project analyzes the **Olist Brazilian E-Commerce dataset** to understand sales performance and revenue trends.
The analysis was performed using **SQL for data analysis** and **Power BI for dashboard visualization**.

The goal of this project is to explore business insights such as revenue performance, order trends, and top-performing cities.

---

  Tools Used

* SQL Server (Data querying and analysis)
* Power BI (Dashboard creation and visualization)
* GitHub (Project documentation and portfolio)

---

   Dataset

The dataset used is the **Brazilian Olist E-Commerce dataset** available on Kaggle.

Main tables used in this project:

* `olist_orders_dataset`
* `olist_order_payments_dataset`
* `olist_customers_dataset`

Key columns used for analysis:

* `order_id`
* `customer_id`
* `order_purchase_timestamp`
* `payment_value`
* `customer_city`

---

   Key Business Metrics

The dashboard focuses on the following important business metrics:

1. **Total Revenue**
2. **Total Orders**
3. **Average Order Value**
4. **Revenue by City**
5. **Monthly Revenue Trend**

---
   Power BI Dashboard

The Power BI dashboard includes the following visualizations:

    1. Total Revenue

Displays the total revenue generated from all orders.

    2. Total Orders

Shows the total number of orders placed.

    3. Average Order Value

Represents the average revenue generated per order.

    4. Total Revenue by City

Bar chart showing the cities that contribute the highest revenue.

    5. Monthly Revenue Trend

Line chart showing how revenue changes across months.

---

   Key Insights

* **Sao Paulo** generates the highest revenue among all cities.
* Revenue trends vary across months, showing seasonal sales patterns.
* Major urban cities contribute significantly to overall revenue.
